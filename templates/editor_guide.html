Editor Guide

Welcome to the Kryptopedia Editor Guide. This resource will help you understand how to create and edit content effectively on our collaborative knowledge platform. By following these guidelines, you'll help maintain the quality and consistency of our articles.

Getting Started

To start contributing to Kryptopedia, you'll need to:

  1. Create an account - Register to get full editing capabilities
  2. Familiarize yourself with our community rules - Understand our standards
  3. Start small - Begin with minor edits before creating full articles
  4. Ask for help - Our community is here to support you

All users can create articles and propose edits, but editors and administrators review changes to maintain quality. As you contribute consistently, you may be promoted to editor status.

Wiki Syntax

Kryptopedia uses a standard wiki markup language. Here are the basics:

Format Wiki Syntax Result
Bold '''Bold text''' Bold text
Italic ''Italic text'' Italic text
Heading (Level 2) == Heading ==

Heading
Internal Link [[Article Title]] Article Title
Internal Link (different text) [[Article Title|Display Text]] Display Text
External Link [https://example.com Example Site] Example Site
Bullet List * Item 1
* Item 2
  • Item 1
  • Item 2
Numbered List # Item 1
# Item 2
  1. Item 1
  2. Item 2
Reference <ref>Citation details</ref> Superscript[1]
Image [[File:example.jpg|thumb|Caption]] [Image with caption]

The Kryptopedia editor includes a toolbar that makes adding these elements easier. You can also use the "Preview" button to see how your markup will appear before saving.

Article Structure

A well-structured article typically includes these elements:

Introduction

Begin with a concise introduction that explains the topic. The introduction should summarize the article without requiring subsections.

Main Content

Organize the main content into logical sections using headings. Use subheadings to create a clear hierarchy:

  • == Level 2 Heading == for main sections
  • === Level 3 Heading === for subsections
  • ==== Level 4 Heading ==== for sub-subsections

References

Include a References section at the end with <references /> to display all references used in the article.

See Also

Optionally include a "See Also" section linking to related articles on Kryptopedia.

External Links

Optionally include relevant external resources for further reading.

Editing Process

There are two ways to contribute to Kryptopedia:

Direct Editing

Registered users can directly edit articles. To do this:

  1. Click the "Edit Article" button on any page
  2. Make your changes in the editor
  3. Add an edit summary explaining your changes
  4. Click "Save Changes" to publish

Proposing Edits

For more substantial changes, or if you're new to Kryptopedia, consider using the proposal system:

  1. Click "Propose Edit" on an article
  2. Make your changes in the editor
  3. Provide a detailed summary explaining your proposal
  4. Submit for review by editors

Editors will review your proposal and either approve or reject it with feedback.

Citations and References

Proper citation is essential to ensure the reliability of our content.

Adding References

To add a reference:

  1. Place the cursor where you want the reference mark to appear
  2. Use the reference button in the editor toolbar or add manually: <ref>Citation details</ref>
  3. Add the citation details in the proper format (see below)
  4. Place <references /> at the end of the article in a "References" section

Citation Formats

For websites:

<ref>cite web|title=Page Title|url=https://example.com|website=Website Name|access-date=2025-04-14</ref>

For books:

<ref>cite book|title=Book Title|author=Author Name|publisher=Publisher|year=Year|pages=xx-xx|isbn=ISBN</ref>

For journal articles:

<ref>cite journal|title=Article Title|author=Author Name|journal=Journal Name|volume=Volume|issue=Issue|year=Year|pages=xx-xx|doi=DOI</ref>

Categories and Tags

Categories and tags help organize content and make it more discoverable.

Categories

Categories group articles by broad topics. When creating or editing an article, add relevant categories in the category field. Use existing categories where possible to maintain organization.

Tags

Tags are more specific keywords that may not fit into broader categories. They help in searching and connecting related content across different categories.

Browse existing categories and tags to understand how content is organized.

Adding Images

Images enhance articles and help explain concepts. To add an image:

  1. First, upload the image using the Upload File page
  2. In your article, use the image syntax: [[File:filename.jpg|thumb|Caption text]]

Additional image parameters:

  • thumb - Creates a thumbnail with caption
  • left, right, or center - Alignment
  • width=300px - Sets the image width

Image Guidelines

  • Only use images you have rights to use (your own, public domain, or appropriately licensed)
  • Provide attribution for images when required by their license
  • Use appropriate file formats: JPG for photos, PNG for graphics, SVG for diagrams
  • Keep file sizes reasonable (typically under 1MB)

Proposing Edits

The proposal system allows for peer review before changes are implemented.

When to Use Proposals

  • Making substantial changes to established articles
  • Adding controversial content that might need discussion
  • When you're new to editing and want feedback
  • When an article is protected or semi-protected

Creating a Good Proposal

  1. Make focused, specific changes rather than overhauling an entire article
  2. Provide a clear, detailed summary explaining why the changes improve the article
  3. Cite reliable sources to support your additions
  4. Be patient while waiting for review

Editors will review your proposal and may suggest improvements before approval.

Reviewing Edits (for Editors)

Editors play a crucial role in maintaining quality by reviewing proposed changes.

Review Guidelines

  1. Check for accuracy and verifiability of information
  2. Ensure proper formatting and wiki syntax
  3. Verify that citations are properly formatted and from reliable sources
  4. Look for neutral, balanced presentation of information
  5. Confirm the edit improves the article

Review Actions

  • Approve - Accept the changes as they are
  • Reject - Decline the changes, providing clear reasons
  • Provide Feedback - Suggest improvements before making a decision

Always be constructive in feedback, focusing on content rather than the contributor.

Best Practices

Follow these principles to contribute effectively to Kryptopedia:

Content Principles

  • Accuracy - Verify information with reliable sources
  • Neutrality - Present information in a balanced, objective manner
  • Clarity - Write in clear, concise language accessible to a general audience
  • Completeness - Cover the topic comprehensively, but avoid unnecessary detail

Editing Tips

  • Make incremental edits rather than overhauling articles at once
  • Use the Preview function before saving changes
  • Include descriptive edit summaries for all changes
  • Check an article's history before making significant changes
  • Discuss controversial changes on the article's discussion page first
  • Be patient with new editors and help them learn

Etiquette

  • Assume good faith when reviewing others' contributions
  • Give constructive feedback rather than just criticizing
  • Acknowledge others' contributions
  • Be open to having your edits modified by others
  • Take breaks if discussions become heated
