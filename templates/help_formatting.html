Kryptopedia Formatting Guide

Introduction to Formatting

Kryptopedia uses a rich text editor that allows you to format your content with various styles and elements. You can use the editor toolbar to apply formatting, or you can directly input HTML if you're familiar with it.

Pro Tip: The editor supports both visual editing and HTML code view. If you prefer to work with raw HTML, click the "Code View" button in the editor toolbar.

Basic Text Formatting

Headings

Use headings to organize your content hierarchically. Kryptopedia supports multiple heading levels:

<h1>Main Title</h1> <h2>Section Title</h2> <h3>Subsection Title</h3> <h4>Minor Section</h4>

Main Title

Section Title

Subsection Title

Minor Section

Note: Each article automatically starts with an h1 title, so it's best to begin your content with h2 headings.

Paragraphs and Line Breaks

Use paragraphs to separate blocks of text, and line breaks for smaller separations:

<p>This is a paragraph of text.</p> <p>This is another paragraph.</p> This is text with a<br>line break.

This is a paragraph of text.

This is another paragraph.

This is text with a
line break.

Text Styling

You can style your text with bold, italic, underline, and more:

<strong>Bold text</strong> or <b>bold text</b> <em>Italic text</em> or <i>italic text</i> <u>Underlined text</u> <s>Strikethrough text</s> <mark>Highlighted text</mark>
Bold text or bold text Italic text or italic text Underlined text Strikethrough text Highlighted text

Lists and Structural Elements

Ordered Lists

Use ordered (numbered) lists for sequences or ranked items:

<ol> <li>First item</li> <li>Second item</li> <li>Third item</li> </ol>
  1. First item
  2. Second item
  3. Third item

Unordered Lists

Use unordered (bullet) lists for items without a specific order:

<ul> <li>Apple</li> <li>Banana</li> <li>Orange</li> </ul>
  • Apple
  • Banana
  • Orange

Nested Lists

Lists can be nested within each other:

<ul> <li>Fruits <ul> <li>Apples</li> <li>Bananas</li> </ul> </li> <li>Vegetables <ul> <li>Carrots</li> <li>Broccoli</li> </ul> </li> </ul>
  • Fruits
    • Apples
    • Bananas
  • Vegetables
    • Carrots
    • Broccoli

Tables

Tables are useful for presenting structured data:

<table> <thead> <tr> <th>Name</th> <th>Age</th> <th>Occupation</th> </tr> </thead> <tbody> <tr> <td>John</td> <td>30</td> <td>Developer</td> </tr> <tr> <td>Sarah</td> <td>28</td> <td>Designer</td> </tr> </tbody> </table>
Name Age Occupation
John 30 Developer
Sarah 28 Designer
Tip: Use the table button in the editor toolbar for an easier way to create and edit tables visually.

Blockquotes

Use blockquotes for quoting external sources or highlighting important sections:

<blockquote> This is a blockquote. It's useful for quoting external sources. </blockquote>
This is a blockquote. It's useful for quoting external sources.

Links and Media

Hyperlinks

Link to external websites or internal pages:

<a href="https://example.com">External link</a> <a href="/articles/bitcoin">Internal link to Bitcoin article</a>
External link Internal link to Bitcoin article
Tip: For internal links to other Kryptopedia articles, you can use the format /articles/article-slug where "article-slug" is the URL-friendly version of the article title.

Images

Include images in your articles:

<img src="/media/example-image.jpg" alt="Description of image"> <figure> <img src="/media/example-image.jpg" alt="Description of image"> <figcaption>Figure 1: Caption for the image</figcaption> </figure>
[Image would appear here]

Note: Before adding images to your articles, you need to upload them to Kryptopedia. Once uploaded, you can copy the image URL to use in your articles.

Code Blocks

For displaying code snippets:

<pre><code>function example() { return "Hello, World!"; } </code></pre>

Use the <code>print()</code> function to output text.

Use definition lists for term-definition pairs:

<dl> <dt>HTML</dt> <dd>HyperText Markup Language, the standard markup language for creating web pages.</dd> <dt>CSS</dt> <dd>Cascading Style Sheets, a style sheet language used for describing the presentation of a document.</dd> </dl>

Content above the line <hr> Content below the line

Best Practices

Consistent Structure

Maintain a consistent structure throughout your articles:

  • Start with an introduction that summarizes the topic
  • Use headings to organize content into logical sections
  • Keep heading levels in proper hierarchical order (h2, then h3, then h4)
  • End with a conclusion or summary section when appropriate

Readability

Enhance readability with these tips:

  • Use short paragraphs (3-5 sentences)
  • Break complex information into lists
  • Use tables to organize structured data
  • Include images to illustrate concepts
  • Use formatting (bold, italic) sparingly for emphasis
Warning: Avoid overusing formatting elements like bold, italic, and highlighting. Too much emphasis can make content harder to read.

Accessibility

Ensure your content is accessible to all users:

  • Include descriptive alt text for all images
  • Use heading levels in proper order for screen readers
  • Ensure sufficient color contrast for text
  • Don't rely solely on color to convey information
  • Use tables appropriately with proper headers
