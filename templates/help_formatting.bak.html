Wiki Formatting Guide

Text Formatting

Format text to emphasize important information or improve readability.

Format Wiki Syntax Result
Bold '''Bold text''' Bold text
Italic ''Italic text'' Italic text
Bold and Italic '''''Bold and italic''''' Bold and italic
Underline <u>Underlined text</u> Underlined text
Strikethrough <s>Strikethrough text</s> Strikethrough text
Superscript <sup>Superscript</sup> x2
Subscript <sub>Subscript</sub> H2O
Code/Monospace <code>Monospace text</code> Monospace text

Formatting Tips:

  • Use bold for first mentions of the article topic and important terms
  • Use italics for terms from other languages, book/publication titles, or emphasis
  • Avoid excessive formatting that can make text harder to read
  • Use underline sparingly as it can be confused with hyperlinks

Paragraphs and Line Breaks

Properly structuring text into paragraphs improves readability.

Paragraphs

To create a new paragraph, leave a blank line between text blocks:

This is the first paragraph.

This is the second paragraph.

This is the first paragraph.

This is the second paragraph.

Line Breaks

To create a line break without starting a new paragraph, use the <br> tag:

This is the first line.<br>
This is the second line in the same paragraph.

This is the first line.
This is the second line in the same paragraph.

Indentation

To indent text, use the colon character:

Normal text.
:Indented once.
::Indented twice.
:::Indented three times.

Normal text.

Indented once.

Indented twice.

Indented three times.

Headings and Sections

Use headings to organize content into logical sections.

Level Wiki Syntax Use For
Level 1 = Heading 1 = Page title (rarely used in the article body)
Level 2 == Heading 2 == Main sections of an article
Level 3 === Heading 3 === Subsections
Level 4 ==== Heading 4 ==== Sub-subsections
Level 5 ===== Heading 5 ===== Rarely needed level
Level 6 ====== Heading 6 ====== Rarely needed level

Heading Tips:

  • Start with Level 2 headings for main sections
  • Don't skip heading levels (e.g., don't jump from Level 2 to Level 4)
  • Ensure headings are descriptive but concise
  • Proper headings automatically generate a table of contents
  • Use sentence case for headings (capitalize only the first word and proper nouns)

Links

Links connect articles and provide references to internal and external resources.

Internal Links

To link to another page within Kryptopedia:

Type Wiki Syntax Result
Basic link [[Article Title]] Article Title
Link with different display text [[Article Title|Display Text]] Display Text
Link to a section [[Article Title#Section Name]] Article Title (links to specific section)
Link to a section with display text [[Article Title#Section Name|Display Text]] Display Text (links to specific section)

External Links

To link to external websites:

Type Wiki Syntax Result
Basic external link [https://example.com] https://example.com
External link with display text [https://example.com Display Text] Display Text

Link Tips:

  • Use internal links to connect related articles
  • Don't over-linkâ€”typically link the first mention of a term in an article
  • Ensure the link text makes the destination clear
  • For external links, prioritize reliable and stable sources
  • Group external links in a dedicated "External Links" section at the end of the article

Lists

Lists organize information in an easy-to-read format.

Bullet Lists

For unordered lists, use asterisks (*):

* First item
* Second item
* Third item
** Indented sub-item
** Another sub-item
* Fourth item
  • First item
  • Second item
  • Third item
    • Indented sub-item
    • Another sub-item
  • Fourth item

Numbered Lists

For ordered lists, use hash symbols (#):

# First item
# Second item
# Third item
## Indented sub-item
## Another sub-item
# Fourth item
  1. First item
  2. Second item
  3. Third item
    1. Indented sub-item
    2. Another sub-item
  4. Fourth item

Definition Lists

For definition lists (term and definition pairs):

; Term 1 : Definition 1
; Term 2 : Definition 2
; Term 3 : Definition 3
Term 1
Definition 1
Term 2
Definition 2
Term 3
Definition 3

List Tips:

  • Use bullet lists for items with no particular order
  • Use numbered lists for sequential items or steps
  • Keep list items parallel in structure (e.g., all start with verbs or all are nouns)
  • Avoid excessive nesting, which can make lists hard to read
  • End each list item with proper punctuation if it's a complete sentence

Tables

Tables organize data into rows and columns.

Basic Table

Create a simple table:

{| class="wikitable"
|+ Caption (optional)
! Header 1 !! Header 2 !! Header 3
|-
| Row 1, Cell 1 || Row 1, Cell 2 || Row 1, Cell 3
|-
| Row 2, Cell 1 || Row 2, Cell 2 || Row 2, Cell 3
|-
| Row 3, Cell 1 || Row 3, Cell 2 || Row 3, Cell 3
|}
Caption (optional)
Header 1 Header 2 Header 3
Row 1, Cell 1 Row 1, Cell 2 Row 1, Cell 3
Row 2, Cell 1 Row 2, Cell 2 Row 2, Cell 3
Row 3, Cell 1 Row 3, Cell 2 Row 3, Cell 3

Table Elements Explained

  • {| - Table start
  • |+ - Caption (optional)
  • ! - Header cell
  • !! - Additional header cell in the same row
  • |- - Row break
  • | - Data cell
  • || - Additional data cell in the same row
  • |} - Table end

Advanced Table Features

Additional table formatting options:

  • Cell spanning: | colspan="2" | Cell spanning two columns
  • Row spanning: | rowspan="2" | Cell spanning two rows
  • Cell alignment: | style="text-align: center;" | Centered text
  • Background color: | style="background-color: #f0f0f0;" | Shaded cell
  • Sortable tables: Add class="wikitable sortable" to make columns sortable

Table Tips:

  • Use tables for data that benefits from columnar organization
  • Include clear headers for each column
  • Keep tables as simple as possible for readability
  • Use the table editor tool in the wiki editor toolbar for easier creation
  • Consider whether the information could be better presented in a different format

Images and Media

Images enhance articles by providing visual information.

Basic Image Syntax

Type Wiki Syntax
Basic image [[File:example.jpg]]
Image with caption [[File:example.jpg|Caption text]]
Thumbnail with caption [[File:example.jpg|thumb|Caption text]]
Resized image [[File:example.jpg|width=300px|Caption text]]
Aligned image [[File:example.jpg|right|Caption text]]

Image Parameters

Common parameters for image formatting:

  • thumb - Creates a thumbnail with caption
  • right, left, or center - Controls image alignment
  • width=300px - Sets the image width (height adjusts proportionally)
  • upright - Adjusts image size to a standard thumbnail size (with thumb)
  • border - Adds a border around the image
  • frameless - Removes the frame but keeps the caption

Galleries

For displaying multiple images in a grid:

<gallery>
File:example1.jpg|Caption 1
File:example2.jpg|Caption 2
File:example3.jpg|Caption 3
</gallery>

Image Tips:

  • Upload images before trying to add them to articles
  • Use descriptive filenames that reflect the image content
  • Add appropriate alt text for accessibility
  • Use thumbnails for most images in articles
  • Keep file sizes reasonable (typically under 1MB)
  • Ensure you have rights to use any images you upload

Templates

Templates are pre-formatted elements that can be inserted into articles.

Basic Template Syntax

{{Template name}}

Templates with Parameters

{{Template name|parameter1=value1|parameter2=value2}}

Common Templates

Template Syntax Purpose
Short Description {{Short description|Brief description}} Adds a brief description at the top of the article
Infobox {{Infobox|title=Title|param1=value1}} Creates a standardized information box
Quote {{Quote|text=Quote text|author=Author}} Formats a blockquote with attribution
Notice {{Notice|type=note|text}} Displays a notice/warning box
Reflist {{Reflist}} Displays the references list

Template Tips:

  • Use the template gallery in the editor toolbar to browse available templates
  • Templates help maintain consistent formatting across articles
  • Check template documentation for required and optional parameters
  • Test templates in the preview mode before saving

References and Citations

References add credibility to your content by citing sources.

Basic Reference

This is a statement that needs a citation.<ref>Author Name, "Article Title", Publication, Date, Page.</ref>

This is a statement that needs a citation.[1]

Named References

For citing the same source multiple times:

First citation.<ref name="smith">John Smith, "Example Article", Journal, 2023, p. 42.</ref>
Later in the text.<ref name="smith" />

First citation.[1]
Later in the text.[1]

Citation Templates

For formatted citations:

<ref>{{cite web|title=Page Title|url=https://example.com|website=Website Name|access-date=2025-04-14}}</ref>

Reference List

To display all references, add this at the end of the article:

== References ==
<references />

Reference Tips:

  • Use references for all factual claims, especially on controversial topics
  • Place reference tags directly after the punctuation they support
  • Use named references to avoid duplicating the same citation
  • Include complete information in citations (author, title, date, source)
  • Use citation templates for consistent formatting
  • Always include a References section at the end of the article

Categories

Categories are now added through the category field in the editor interface. Historically, they could be added with:

[[Category:Category Name]]

Advanced Formatting Tips:

  • Use advanced formatting sparingly and only when it enhances the content
  • Test advanced features in the preview mode to ensure they render correctly
  • Consider whether simpler formatting could achieve the same purpose
  • Check the help documentation for details on specialized formatting options
