Adding Media to Kryptopedia Articles

Introduction to Media Files

Media files such as images, diagrams, and charts can greatly enhance Kryptopedia articles by making complex concepts more accessible, providing visual references, and improving overall comprehension. This guide will help you understand how to properly add and use media in your articles.

Tip: Well-chosen media can significantly improve an article's effectiveness. Images should complement your text and add value rather than simply decorating the page.

Types of Media Supported

Kryptopedia supports several types of media files:

File Type Extensions Best Used For
Images .jpg, .jpeg, .png, .gif Photographs, screenshots, diagrams, logos
Vector Graphics .svg Logos, icons, diagrams that need to scale
Documents .pdf Technical papers, specifications, detailed documentation
Note: There is a file size limit of 10MB per upload to ensure optimal site performance. Please optimize your files before uploading.

Uploading Media Files

The Upload Process

Before you can add media to an article, you need to upload it to Kryptopedia. Here's the workflow:

1
Prepare your file: Ensure your file is in a supported format and optimized for web use (compressed, appropriate dimensions).
2
Navigate to upload page: Click on "Upload File" in the sidebar navigation or go to the upload page directly.
3
Select and upload your file: Click the "Choose File" button, select your file, and then click "Upload".
4
Note the file URL: After upload, you'll receive a URL for your file. Copy this for use in your article.
Tip: Choose descriptive filenames that reflect the content of the file before uploading. This helps with organization and searchability.

File Naming Best Practices

Good file names are important for organization and clarity:

  • Use descriptive names that indicate the content (e.g., "ethereum_blockchain_diagram.png" rather than "image1.png")
  • Avoid spaces in filenames; use underscores or hyphens instead
  • Keep filenames reasonably short but descriptive
  • Use lowercase letters to avoid case-sensitivity issues

Adding Media to Articles

Basic Image Insertion

Once you've uploaded an image, you can add it to your article using the editor's image insertion tool or by adding the appropriate HTML:

<img src="/media/your_filename.jpg" alt="Descriptive text about the image" width="500">

The key components are:

  • src: The path to your image (provided after upload)
  • alt: Alternative text that describes the image (important for accessibility)
  • width: Optional attribute to specify image size (maintain aspect ratio)
Important: Always include meaningful alt text with your images. This helps users with visual impairments and improves article accessibility.

Image Placement and Sizing

Consider the following for optimal image placement:

  • Context: Place images near the relevant text they illustrate
  • Spacing: Add appropriate margins around images for readability
  • Size: Adjust image size to balance detail with page layout
  • Alignment: Align images to complement the text flow

To center an image with a caption:

<figure style="text-align: center;"> <img src="/media/your_filename.jpg" alt="Descriptive text" width="500"> <figcaption>Your caption text here</figcaption> </figure>

Media Best Practices

Image Optimization

For optimal performance and user experience:

  • File Size: Keep images under 500KB when possible
  • Dimensions: Resize images to appropriate dimensions before uploading
  • Format: Use appropriate formats for content type:
    • JPEG: For photographs and complex images with many colors
    • PNG: For screenshots, diagrams, and images requiring transparency
    • SVG: For diagrams, charts, and logos that need to scale
    • GIF: For simple animations (use sparingly)
  • Compression: Use image compression tools to reduce file size without significant quality loss

Copyright and Attribution

Always ensure you have the right to use and share the media you upload:

  • Only upload media you created yourself, that is in the public domain, or that has a license permitting reuse
  • Provide proper attribution for any media that requires it
  • Include source information in the image caption or in the article text
Warning: Uploading copyrighted material without permission can lead to its removal and may have legal consequences. When in doubt about the copyright status of media, do not upload it.

Accessibility Considerations

Make your media accessible to all users:

  • Always include descriptive alt text for images
  • Use captions to provide context for figures and diagrams
  • Ensure sufficient color contrast in diagrams and charts
  • Don't rely solely on images to convey critical information

Good alt text should be concise but descriptive. For example:

  • Poor: "Graph"
  • Better: "Graph showing Bitcoin price trend from 2010 to 2023"
